# 15. 3Sum

## ğŸ“˜ Problem Statement
Given an integer array `nums`, return all the **unique triplets** `[nums[i], nums[j], nums[k]]`
such that:
- `i != j`, `i != k`, and `j != k`
- `nums[i] + nums[j] + nums[k] == 0`

The solution must not contain duplicate triplets.

## ğŸ§  Brute Force Approach

### ğŸ”¹ Idea
Try all possible triplets using three nested loops and store unique triplets using a Set.

### ğŸ”¹ Algorithm
1. Use three nested loops for all `(i, j, k)` combinations
2. Check if their sum is zero
3. Sort each triplet before adding to Set (to avoid duplicates)

### â± Complexity
- **Time Complexity:** O(nÂ³)
- **Space Complexity:** O(n)

### ğŸ’» Java Code (Brute Force)
```java
import java.util.*;

class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Set<List<Integer>> set = new HashSet<>();
        int n = nums.length;

        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                for (int k = j + 1; k < n; k++) {
                    if (nums[i] + nums[j] + nums[k] == 0) {
                        List<Integer> triplet = Arrays.asList(nums[i], nums[j], nums[k]);
                        Collections.sort(triplet);
                        set.add(triplet);
                    }
                }
            }
        }
        return new ArrayList<>(set);
    }
}
````

## ğŸ§  Better Approach (HashSet)

### ğŸ”¹ Idea

Fix one element and use a HashSet to find the remaining two elements.

### ğŸ”¹ Algorithm

1. Loop index `i` from 0 to n
2. Use HashSet to find pairs with sum `-nums[i]`
3. Use another Set to store unique triplets

### â± Complexity

* **Time Complexity:** O(nÂ²)
* **Space Complexity:** O(n)

### ğŸ’» Java Code (Better Approach)

```java
import java.util.*;

class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Set<List<Integer>> result = new HashSet<>();

        for (int i = 0; i < nums.length; i++) {
            Set<Integer> seen = new HashSet<>();

            for (int j = i + 1; j < nums.length; j++) {
                int complement = -nums[i] - nums[j];

                if (seen.contains(complement)) {
                    List<Integer> triplet = Arrays.asList(nums[i], nums[j], complement);
                    Collections.sort(triplet);
                    result.add(triplet);
                }
                seen.add(nums[j]);
            }
        }
        return new ArrayList<>(result);
    }
}
```

## ğŸ§  Optimal Approach âœ… (Sort + Two Pointer)

### ğŸ”¹ Idea

Sort the array. Fix one element and use two pointers to find pairs.
Skip duplicates carefully.

### ğŸ”¹ Algorithm

1. Sort the array
2. Fix index `i` and skip duplicates
3. Use `left` and `right` pointers
4. Adjust pointers based on sum
5. Skip duplicates after finding valid triplet

### â± Complexity

* **Time Complexity:** O(nÂ²)
* **Space Complexity:** O(1) (excluding output)

### ğŸ’» Java Code (Optimal Solution)

```java
import java.util.*;

class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums);

        for (int i = 0; i < nums.length - 2; i++) {

            if (i > 0 && nums[i] == nums[i - 1]) continue;

            int left = i + 1;
            int right = nums.length - 1;

            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum == 0) {
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));

                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;

                    left++;
                    right--;
                } 
                else if (sum < 0) {
                    left++;
                } 
                else {
                    right--;
                }
            }
        }
        return result;
    }
}
```
